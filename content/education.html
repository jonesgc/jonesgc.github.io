Computing at UWE entails more than learning theoretical aspects of: Programming (C, C++,PHP, JAVA, Python and more), System design, Software Engineering practices and network systems; but an exploration into how they are applied in the real world through a range of projects designed to encourage skills that can be transferred into the professional world. Unless otherwise stated all modules had an examination as part of the assessment for that module. Additionally these modules listed were the ones I personally did, in the third year we could select which two modules to do out of five choices.

Year 3
Advanced Topics in Web Development 1

Research and Build a Prototype Implementation: The focus of this assignemnt was a currency converter. This required the use of an external API then the creation of our own service which could provide currency conversions to a website. We were mandated to use XML as part of the functionality for this project, in addition to PHP, HTML, JS etc. The use of XML was to store the values returned from the external APIs and allow our own functionality to use it as a source.

Advanced Databases

Coursework Based on the Lab Experiences: Over the course of this module we undertook a range of tasks in the tutorial sessions, such as implementing views on an SQL Database. All these topics were then included in some way into the Coursework which required us to individually perform similar tasks to what we did in the tutorial but often to a greater degree of complexity. This included performing a normalisation process on a case study and the generation of a functional Databases design on the senario. In addition to SQL this coursework also included a segment on NoSQL, which we used MongoDB, were the requirement was to translate the SQL design into a NoSQL solution.

Mobile Applications

Assignment: Using the Companies House API we had to produce an Android app which allowed the user to search for a Company in some manner then present the returned data in some form of graph related diagram. Marks were awarded for good practice and general quality of the app.

Ethical and Professional issues in Computing and Digital Media

Individual Report: Focusing on the Ethical issues that might be present in a certain area of Computing of our choosing, for example the Ethical Implications of facial Recognition in smart cities. This report had to be needed to be current to a degree which means it had to touch on GDPR which was a large focus of this module on this year, since it is was 2018 that GDPR came into effect. Our choice of topic also had to relate to members of a small team for the other part of the assessment for this module the Group Presentation.

Group Presentation: In teams of around 4 we had to research an area, in which our Individual Report should have focused on an area. Then from this research build a presentation on the Ethical and Professional Issues in this area, this meant often referencing the code of ethics (were available) for relevant organisations like IEEE, ACM and the examination of legislation such as GDPR.

Requirements Engineering

Essay: Requiring research into one of a selection of disasters of which software issues were identified to one of the main factors, for example the 1996 Ariane 5 failure. The research and resulting essay was to be focused on the failures in the Requirement Engineering process that led to or contributed to the disaster.

Computing Project

Report (Disseration): The Computing Project module is the largest module in terms of number of credits available, additionally it was the only module to run the course of both final semesters in the final year. The assignment can be divided into two elements, the the report and the software created as a result of the research for the report and used in the demonstration assignment of this module. The report element was on a topic of your own choosing, with input from a Supevisor relating to topic area for example if the topic chosen was on Web Technologies then the supevisor is ideally a lecturer with knowledge in that area. After the topic is selected work on the research for the project could beign, this research had be focused mainly on acedemic sources in reputable publications such as IEEE. The remainder of the report would be on the design of the software being created as a result of your research and the methodology behind your process. This project incorporated all of the elements taught in the other modules in all three years of the course.

Demonstration: This demonstration was conducted to your project supevisor and a seccond marker. Focused on displaying the functionality of the software you had created and asnwering questions on the software and the report for your project.

Year 2
Data, Schemas and Aplications

Completion of Worksheets and Online Tutors: Series of tasks aimed at teaching key concepts of Web Development and Data related tasks, such as use of the XMl ecosystem.

Data Modeling, Integration and Presentation Task: Working in a small team of 4 persons, this assignement required the creation of a travel booking Website that incorporated elements of php, Java Script, HTML and XML. As part of this assignment it required an individual component based on the group project, for example the creation of a Flikr plugin that could be integrated into the main project. At the end of the assignment each team was required to present their work to the module lecturer and instructors.

Software Engineering

Group Project Proposal: After investigation of various topics of software engineering and design as a group we had to create a project proposal to create a peiece of software, this included elements such as planned software development method which for this module was the Raitional Unified Process. This proposal was linked to the other two assessment items of this module.

Group Based and Individual Project Deliverables: For the individual element of this assignment each member of the group was required to create a Use Case for the project, complete with Use Case specification and associated diagrams. Additionally as a group we had to follow our created project plan and produce some form of software, this was done using the Java Programming language.

Individual Reflective Report: After completion of the group based assessment tasks we had to create a reflective report on our personal and group performance in the other assignments for this module.

Operating Systems

Log Book: Series of tasks introducing Operating Systems. The primary components of this asessment were on Pintos and required implementing a program stack. Then once the stack was impletmented a set of system calls were required to be implemented, such as wait or exit syscalls. This module also required the use of a Linux virtual machine as a development environment.

Internet of Things

This module did not have an examination.

Log book: This consisted of three tasks that needed to be completed. These tasks centred about the use of the BBC Microbit to teach IoT concepts. The first task was an introduction into the BBC Microbit and as such it required the creation of some form of game on the Microbit, this helped introduce the hardware we had available on the Microbit such as the accelerometer. The seccond task was to pass some form of messages from one Microbit to another, with the requirement that the message be encrypted in some form. This also required the user to be able to input a message on one microbit then see it displayed on the other connected microbit, this task used a wired connection between the microbits. The third task required using a wireless connection method to perform a similar function as the seccond task. An option was given in the thrid task to use Blue Tooth Low Energy to do something.

Presentation: Requiring research into an aspect of IoT, such as use of IoT in smart cities, then the creation of a written essay and a presentation to a lecturer and some peers on a topic of your choice relating to the essay.

C++ Development

Portfolio tasks: The overral goal of this part of this assessment was to create a version of the Lorenz machine in C++. This assessment also introduced some software engineering tasks, such as the creation of a class diagram and then attempting to create a functioning program from the diagram. This module was the first real introduction into Object Oriented Programming (OOP) on the computing course. The assignment comprised of 5 tasks, each task built from the previous one gradually building functionality into the overrall program.

Year 1
Web Programming

Individual Worksheet Assessments: This part of the module required the completion of a small worksheet within a tutorial session (2hrs). Overall there were five worksheets to complete covering the major topics being taught. These topics were: Python, Python CGI, HTML + CSS + JS, and SQL.

Group Presentation: In groups of four we were set a project, to create a travel booking website. This entailed creating a database in SQL using phpmyadmin, the front end in HTML + CSS + JS and the server side scripts in Python. This was assessed with a group presentation and a review of our submitted code.

Programming in C

Duck Shoot Game: Working on 32bit Altera boards, using switches + a button for input, Seven Segment digital display + LED's for output. We had to develop a "Duck shoot game" which required an alternating pattern of led's moving in a direction, which could be changed via a switch, and "shooting" them at a specific position via a button. This would then increment the score on the seven segment display. This was assessed on a demonstration of correct logic and code quality.

MARCO Robot: Development of three programs: a line follower using the robots two motor wheels + floor sensors, room escape using the motor wheels + "bumper" sensors and light follower using the motor wheels + light sensor mounted on a stepper motor. This was assessed on demonstration of correct logic for each task in addition to code quality.

Business Applications

Investigation, report and Presentation: In groups of four a case study and interview was used as the basis for an investigation into the requirements of a fictional company. After the investigation we were required to create a proposal for a new database system for the company, then present our findings using Entity Relation Diagrams for the proposed system. Assessed on individual and group performance in the presentation in addition to the Entity Relation Diagram(s).

Prototype and Group Demo: Following from the first assignment in this module, in the same groups, required the transition from theoretical Entity Relation Diagrams to a working data base, in phpmyadmin using sql. Assessed on demonstration of database in addition to knowledge of SQL and theoretical aspects of the system.

Principles Of Computing

E assessment: This section focused on computing related mathematical concepts such as set theory, these assessments were done on-line via the university e - assessment tool.

Written Assessment: Required the creation of diagrams and completion of work sheets relating to subject matter being delivered at the time, example being creation of a state machine diagram for a basic calculator.
